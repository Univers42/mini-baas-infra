⚡ Dual Data Planes Demo

PostgreSQL relational + MongoDB documents through unified gateway. Same auth, different schema models.

🗄️ PostgreSQL: Relational Schema

Collection: public.mock_orders

Schema: Structured table with constraints

  • id: UUID (primary key)
  • owner_id: TEXT (RLS enforced)
  • order_number: TEXT (unique constraint)
  • total_cents: INT (check ≥ 0)
  • status: ENUM (pending, paid, cancelled)
  • created_at, updated_at: TIMESTAMP

📄 MongoDB: Document Schema

Collection: mock_catalog

Schema: JSON document validator

  • owner_id: string (RLS enforced)
  • sku: string (2-64 chars)
  • name: string (2-120 chars)
  • category: string
  • price_cents: int (≥ 0)
  • tags: array of strings (optional)
  • in_stock: boolean (optional)
